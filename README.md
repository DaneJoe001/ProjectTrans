## cpp_project_trans

一个以“网络框架”为核心的个人项目。在自研的跨平台网络基础设施之上，提供示例性的客户端与服务端（含可唤醒的 Qt 面板），用于验证传输、存储与交互能力，并为未来客户端功能扩展打底。

### 核心理念
- 网络框架优先：抽象套接字与事件循环，关注连接管理与数据传输的稳定性与可移植性
- 客户端/服务端只是示例载体：展示框架用法与端到端链路
- 易扩展的客户端：后续以插件化/模块化方式快速添加新能力
- 个人维护：演进节奏以可读性和稳定性优先

### 主要特性
- 事件驱动网络：epoll 事件循环、POSIX 套接字（后续统一抽象屏蔽平台差异）
- 持久化与管理：SQLite 存储文件信息；简洁日志组件
- 客户端与服务端示例：验证上传/下载与元数据记录
- GUI 面板（可选）：用于查看连接/文件信息；可隐藏并在需要时唤醒

### 目录概览
- `source/common/`：通用网络/数据库/日志（网络框架核心在此演进）
- `source/server/`：服务端入口、业务仓储、面板（Qt Widgets）
- `source/client/`：客户端入口与基础交互

### 构建
依赖：CMake 3.20+、C++20、Qt 6（Core/Gui/Widgets）、OpenSSL、SQLiteCpp。
```bash
cmake -Bbuild --preset gcc-debug
cmake --build build
```
生成可执行文件：`server`、`client`。

### 运行
- 服务端：
```bash
./server
```
- 客户端（可选）：
```bash
./client
```

### 面板唤醒（简述）
- 面板默认随进程启动显示，关闭仅隐藏，不影响服务端线程
- 可通过信号触发唤醒（如 SIGQUIT）：信号置位 → GUI 线程定时器检测 → `show/raise/activateWindow`

### 路线与扩展
- 网络层：统一平台抽象（如套接字选项/事件模型），完善错误处理与可观察性
- 客户端：引入模块化/插件化接口，方便新增业务插件与 UI 组件
- 服务端：管理接口与权限控制，便于外部工具集成
